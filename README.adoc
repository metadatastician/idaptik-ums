= idaptik-ums
:toc:

The Unified Modding Studio. Procedural-generation pipeline for idaptik
characters, levels, objects, and DLC extensions.

== Architecture

----
                       AffineScript shell
                              │
                              ▼
                       Idris2 ABI (17 modules)
                              │
                              ▼
                       Zig FFI (11 files)
                              │
                              ▼
                       Gossamer (host runtime)
----

* *Shell* — AffineScript port of the existing ReScript
  `idaptik-ums/src/{App.res, editor/, abi/}` tree.
* *ABI* — Idris2 dependent-type proofs of the shell↔FFI boundary
  (17 modules, 0 believe_me — STATE.a2ml authoritative).
* *FFI* — Zig C-ABI exports (11 files; see `ffi/zig/src/`).
* *Host* — Gossamer (NOT Tauri — the Tauri references in the current
  monorepo's docs are stale; see Recovery PR 4).

== UMS↔game integration (most critical)

UMS produces generated content — characters, levels, objects, DLC — as
`dlc-manifest.json` artifacts in the same shape as AssetPack output.
The game's existing DLC loader consumes them with no game-side code
changes.

This is the smallest possible bridge that unlocks the procedural-gen
pipeline. The contract is declared in `idaptik-contracts/contracts.a2ml`
under DeviceKind/Zone (today) and extended as new procedural generators
land.

== ProvenBridge

`ProvenBridge.idr` (currently 2 typed holes + commented `proven`
dependency) is gated to `experimental/` per Recovery PR 4. Decision to
fully wire the proven dependency or fold the holes via AffineScript
effect rows is deferred until the Wave 1 VM port surfaces compiler
maturity signal.
