= Universal Modding Studio

Universal Modding Studio (UMS) is an independent authoring, generation,
validation and packaging platform for game worlds, systems, agents,
narratives and rules.

UMS is an authoring environment, modelling substrate, constraint-driven
generation and transformation system, validator, packager, simulation-preview
host through adapters, and SDK for game profiles. It is not IDApTIK, the
Enaction Engine, a renderer required by released games, or a universal game
runtime. Released games consume compiled packages; they do not depend on the
studio application.

IDApTIK is the first and deepest profile and the immediate product focus.
Chronicles of Slavia is the bounded second profile used to show that the
abstractions travel without importing either game's ontology into the core.
Its authoritative game design currently lives in the separate
`metadatastician/chronicles-of-slavia` repository.

== What exists now

This is a staged migration that preserves the repository's formal work and
history:

* `crates/ums-profile-sdk` implements the versioned `ums-profile/1` typed and
  reflective profile contract.
* `profiles/idaptik` owns generated reflection over the established Nickel
  vocabularies, edit verbs, component model, constraints and package/runtime
  declarations.
* `profiles/chronicles-of-slavia` provides a minimal first-zone reflection and
  fixture. It has no loader or runtime integration yet.
* `crates/ums-ai-edit` is the tested miniKanren edit/generation engine. Its
  unqualified API is retained as an IDApTIK compatibility facade.
* `crates/ums-dlc` validates package envelopes, existing DLC, capability
  declarations, edit scripts and IDApTIK payload contracts.
* `config` remains the Nickel source of truth for the working IDApTIK closed
  worlds and verbs; `just gen-check` detects generated drift.
* `abi` contains the Idris2 ABI work and `ffi/zig` the Zig C-ABI bridge.
* `dlc` contains the current package fixtures and reversible VM lineage.

There is no interactive studio frontend in this tree. The current interfaces
are command-line and LLM-driven. The multi-lens application target is
specified in `docs/STUDIO-INTERFACE.adoc`.

== Architecture boundary

The host-neutral vocabulary is deliberately narrow: stable identities,
components, state, events, space/time, agency, perception and knowledge,
goals, relationships, affect, narrative relationships, asset references,
constraints, tests, provenance, edit history and package capabilities.

PBX systems, anti-hackers, keycards, remedies, security tiers, the Rift and
living taxis are profile concepts. A profile publishes data that a future UI
can inspect and implements typed service contracts where executable behaviour
exists. UMS does not dynamically execute arbitrary plugin code named by a
manifest.

See `ARCHITECTURE.md`, `docs/PROFILE-SDK.adoc` and
`docs/ENACTION-ADAPTER.adoc`.

== Formal and verification posture

UMS retains immutable state-to-state edit relations, replayable edit scripts,
miniKanren and finite-domain generation, refusal rather than fabrication,
Nickel registries and schema generation, runtime validation, Idris2 ABI work,
Zig FFI, DLC packaging and claim-to-evidence documentation.

Guarantees are labelled as machine-checked, runtime-validated, tested,
designed or aspirational. Relational runtime checks are not described as
machine-checked proofs. The full UMS → profile compiler → released-game loader
round trip is not yet complete; `docs/ROUNDTRIP-STATUS.adoc` states the exact
remaining seam.

== Development

[source,sh]
----
just gen-check
cargo fmt --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
just dlc-check
just test-all
----

`just test-all` also exercises the Zig and Idris2 gates when their pinned
toolchains are available. Do not hand-edit generated profile JSON or the
generated IDApTIK registry.

== Repository identity

The canonical product name is *Universal Modding Studio*, short name *UMS*,
with desired GitHub slug
`metadatastician/universal-modding-studio`. This repository is renamed in
place when GitHub authentication and permissions permit it; history is never
recreated or copied.
